# スキャルピングトレード 実行ワークフロー

**目的**: 感情を排除し、体系的・再現性の高いトレードを実現

---

## 📋 トレード実行の全体フロー

```
1. プレトレード準備（5分）
   ↓
2. 市場分析（10分）
   ↓
3. エントリー判断（5分）
   ↓
4. ポジション管理（リアルタイム）
   ↓
5. 決済判断（即座）
   ↓
6. ポストトレード記録（5分）
```

---

## 🔍 PHASE 1: プレトレード準備（5分）

### 1-1. 精神状態チェック
```
✅ チェックリスト:
□ 十分な睡眠を取った（6時間以上）
□ 感情的になっていない（怒り・焦り・興奮なし）
□ 集中できる環境（静かな場所、中断なし）
□ 連続損失中でない（3連敗以内）
□ トレード資金に余裕がある
```

**❌ 1つでも該当しない場合 → 本日トレード中止**

### 1-2. システム準備
```
□ Bybit MCP起動確認
□ TradingViewチャート起動
□ トレード記録ファイルを開く
□ 損益計算機を準備
□ タイマー設定（30分）
```

### 1-3. 資金・リスク確認
```
現在資金: ¥______
1トレード最大リスク: ¥______ (資金の2%)
最大ポジションサイズ: ¥______
本日許容損失: ¥______ (資金の6%)
本日トレード済み回数: ___ / 10回
```

---

## 📊 PHASE 2: 市場分析（10分）

### 2-1. Bybit MCPでマルチタイムフレーム分析

**実行コマンド例**:
```
BTC/USDTのマルチタイムフレーム分析を実行
- 15分足、1時間足、4時間足
```

**確認項目**:
```
□ 15分足: トレンド方向 [上昇/下降/レンジ]
□ 1時間足: トレンド方向 [上昇/下降/レンジ]
□ 4時間足: トレンド方向 [上昇/下降/レンジ]
□ ML-RSI（15分）: ____ [買われすぎ/売られすぎ/中立]
□ ボラティリティ: [高/中/低]
□ 出来高: [平均以上/平均以下]
```

### 2-2. 重要レベル特定
```
直近サポート: $______
直近レジスタンス: $______
リクイディティゾーン: $______ - $______
```

### 2-3. トレード可否判断
```
✅ トレード可能条件:
□ ボラティリティが中以上
□ 明確なトレンドまたはレンジ
□ 出来高が平均以上
□ 重要経済指標発表まで30分以上

❌ 以下の場合はトレード見送り:
□ ボラティリティが極端に低い
□ 方向感がない
□ 重要イベント直前
□ スプレッドが広い
```

## 🎯 PHASE 3: エントリー判断（5分）

### 3-1. セットアップ確認

#### ロングエントリー条件
```
必須条件（全て✅で初めてエントリー検討）:
□ 4時間足: 上昇トレンドまたは中立
□ 1時間足: 上昇トレンド
□ 15分足: 直近で上昇の兆し
□ ML-RSI (15分): 30以下から反発、または40-50
□ 価格: サポートライン付近
□ 出来高: 増加傾向
□ ローソク足: 陽線確定
```

#### ショートエントリー条件
```
必須条件（全て✅で初めてエントリー検討）:
□ 4時間足: 下降トレンドまたは中立
□ 1時間足: 下降トレンド
□ 15分足: 直近で下降の兆し
□ ML-RSI (15分): 70以上から反落、または50-60
□ 価格: レジスタンスライン付近
□ 出来高: 増加傾向
□ ローソク足: 陰線確定
```

### 3-2. リスクリワード計算

```
エントリー価格: $______
損切り価格 (-1.5%): $______
利確目標1 (+0.5%): $______
利確目標2 (+1.0%): $______

リスクリワード比: 1 : ____ (最低1:2必須)
```

**❌ リスクリワード比が1:2未満 → エントリー見送り**

### 3-3. ポジションサイズ計算

```
計算式:
ポジションサイズ = (資金 × 2%) / 損切り幅(1.5%)

例: 資金¥30,000の場合
ポジションサイズ = (30,000 × 0.02) / 0.015 = ¥40,000相当

実際のポジションサイズ: ¥______
```

### 3-4. 最終チェックリスト

```
エントリー直前の確認:
□ 全てのエントリー条件を満たしている
□ リスクリワード比 1:2以上
□ ポジションサイズを計算済み
□ 損切りラインを設定済み
□ 利確目標を設定済み
□ 本日の許容損失を超えない
□ 感情的になっていない

✅ 全てOK → エントリー実行
❌ 1つでもNG → エントリー見送り
```

## 🎮 PHASE 4: ポジション管理（リアルタイム）

### 4-1. エントリー直後
```
□ 損切り注文を即座に設定（-1.5%）
□ 利確目標1を設定（+0.5%）
□ 利確目標2を設定（+1.0%）
□ タイマー開始（最大保有30分）
□ エントリー理由をメモ
```

### 4-2. ポジション監視

**5分ごとにチェック**:
```
□ 想定シナリオ通りに動いているか
□ ML-RSIの変化
□ 出来高の変化
□ ローソク足パターン
```

**異常検知 → 即撤退**:
```
❌ 以下の場合は損切りラインに到達前でも決済:
□ 想定と逆のトレンド発生
□ 出来高急減
□ 重要ニュース発表
□ システムエラー・遅延
```

### 4-3. 利確ルール

```
利確目標1達成（+0.5%）:
→ 50%決済 + 損切りラインを建値に移動

利確目標2達成（+1.0%）:
→ 残り全決済

30分経過:
→ 含み益でも全決済（時間切れルール）
```

## ⚠️ 緊急時対応

### トラブル発生時
```
□ システムエラー → 全ポジション即決済
□ ネット切断 → 再接続後、状況確認して判断
□ 急激な価格変動 → 損切り優先
□ フラッシュクラッシュ → ポジション保有しない
```

### ルール違反時
```
□ 感情的エントリー → 即決済 + 本日終了
□ ポジションサイズ超過 → 即一部決済
□ 損切り未設定 → 即設定または決済
```

## 🎯 成功の10原則

1. **ルールを100%守る** - 1度の例外が習慣を崩す
2. **感情を排除** - 機械的に実行
3. **記録を徹底** - 全トレード記録必須
4. **小さく勝つ** - 0.5-1%の利益を積み重ねる
5. **損切りを守る** - -1.5%で必ず切る
6. **リスク管理優先** - リターンは後からついてくる
7. **過信しない** - 2連勝後は特に注意
8. **休憩を取る** - 疲労は判断力を下げる
9. **振り返る** - 毎トレード後に必ず記録
10. **継続する** - 1ヶ月で判断しない
