# Code Style & Conventions
- **Language**: TypeScript targeting ES modules (`type": "module"`).
- **Structure**: CLI-driven functional modules; keep exports typed using shared interfaces from `src/types`.
- **Linting**: ESLint configured via `npm run lint`; follow existing camelCase identifiers and descriptive function names.
- **Formatting**: No formatter enforced; align with 2-space indentation pattern found in repo.
- **Comments**: Prefer concise inline comments only when necessary, consistent with Japanese annotations in repo docs.
