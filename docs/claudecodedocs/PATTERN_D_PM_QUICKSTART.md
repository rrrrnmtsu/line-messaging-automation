#### ステップ2: データ構造設計（30分）

**Claude Codeに依頼**:
```
週次レポート自動生成システムを作りたいです。

【現状】
- 毎週金曜にレポートを手作業で作成（2時間かかる）
- レポート内容:
  1. 今週の完了タスク一覧
  2. 進行中タスクの状況
  3. 来週の予定
  4. 課題・ブロッカー

【使用ツール】
- Notion（タスク管理）
- Slack（レポート送信先）

【やりたいこと】
1. Notionからタスクデータを自動取得
2. レポート形式に整形
3. Slackに自動投稿

データ構造とシステム設計を提案してください。
```

### 1. 小さく始める

### 2. 実務で使う
- 作るだけでなく、実際に使う
- 毎週レポートを自動生成
- 効果を数値で測定

### 3. 横展開する
- 週次レポートの仕組みを月次レポートにも
- 同じパターンで他の業務も自動化
- プロンプトをテンプレート化

### 4. チームに共有
- 自分だけでなく、チームにも展開
- ドキュメント化して引き継ぎ可能に
- 改善提案を集める

## 🚨 よくある質問

### Q1: Notionを使っていないけど大丈夫？
**A**: Trello、Asana、Jira、Google スプレッドシートでも同様に実装可能です。Claude Codeに「[ツール名]で実装してください」と指定すればOK。

### Q2: プログラミングの知識がないけど大丈夫？
**A**: Claude Codeが全部書いてくれます。あなたは「何をしたいか」を伝えるだけ。

### Q3: エラーが出たらどうする？
**A**: エラーメッセージをそのままClaude Codeに貼り付けて「このエラーを修正してください」と依頼。

### Q4: もっと高度なことをしたい
